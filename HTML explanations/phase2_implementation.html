Phase 2: Data Models Implementation - Complete

For Phase 2 of the Gym Management System project, we have created the necessary data model classes to support the new database tables introduced in Phase 1.

Completed Components

1. Data Model Classes

We have implemented the following data model classes:

1.1. EquipmentData Class

  • File: src/gymmanagementsystem/equipmentData.java
  • Description: Represents equipment items in the gym
  • Properties:
    • id (Integer)
    • equipmentId (String)
    • name (String)
    • type (String)
    • purchaseDate (Date)
    • lastMaintenance (Date)
    • nextMaintenance (Date)
    • status (String)

1.2. ScheduleData Class

  • File: src/gymmanagementsystem/scheduleData.java
  • Description: Represents class schedules
  • Properties:
    • id (Integer)
    • scheduleId (String)
    • className (String)
    • coachId (String)
    • dayOfWeek (String)
    • startTime (Time)
    • endTime (Time)
    • maxCapacity (Integer)
    • currentEnrollment (Integer)
    • status (String)
  • Additional methods:
    • isFull() - Checks if class has reached capacity
    • getAvailableSlots() - Returns number of available slots
    • getTimeRange() - Returns formatted time range

1.3. PaymentData Class

  • File: src/gymmanagementsystem/paymentData.java
  • Description: Represents payment transactions
  • Properties:
    • id (Integer)
    • paymentId (String)
    • memberId (String)
    • amount (Double)
    • paymentDate (Date)
    • paymentMethod (String)
    • description (String)
  • Additional methods:
    • getFormattedAmount() - Returns amount formatted as currency
